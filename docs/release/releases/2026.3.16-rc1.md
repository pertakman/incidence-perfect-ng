## What Changed (Highlights)

- Touch UI Step 2 completed:
  - `Advanced` layout with full control row
  - `Simple` layout with large `ZERO` + `VIEW` buttons
  - touch-only layout switching by long hold
- Hold-tip guidance improved:
  - countdown updates during hold
  - `VIEW` tip bands clarify `AXIS`, `MODE`, and `SWITCH UI`
- Readability and usability tuning:
  - larger touch targets
  - larger readout text
  - reduced flicker/overdraw artifacts around readout/button boundary
- Persistence improvements:
  - touch layout mode persists across reboot
  - axis display mode persists across reboot
- ZERO workflow timing update:
  - stillness detection remains `1000 ms`
  - averaging window adjusted to about `1000 ms` (`50` samples at `~20 ms`)
- Web UI parity updates:
  - control row reordered to `FREEZE`, `ZERO`, `OFFSET CAL`, `AXIS`, `MODE`, `ALIGN`, `ROTATE`, `SLEEP`
  - readout smoothing added for steadier live web values

## OTA Notes (For This RC)

- OTA gate requires target version to be newer than current unless `Force` is enabled.
- If your device already runs `2026.3.16`, same-version OTA is expected to be rejected with `Force` unchecked.
- For normal positive-path OTA verification:
  - update from an older firmware, or
  - build a newer version (for example `2026.3.17`) and use its matching SHA-256.
